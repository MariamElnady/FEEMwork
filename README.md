# Fluorescence Excitation Emission Matrices (FEEM)

## Overview

Fluorescence excitation emission matrices (FEEMs) provide a rapid and sensitive method for characterizing dissolved organic matter (DOM) in water samples. Quantitative analysis is commonly performed using fluorescence regional integration (FRI), fluorescence indices such as the humification index (HIX) and biological index (BIX), and parallel factor analysis (PARAFAC). These methods can be used to track changes in NOM composition across drinking water treatment processes, source waters, and watershed systems.

## Reading FEEM Files

Use the `read_feem()` function from the `cwrshelpr` package to import and clean FEEM data files generated by the Horiba Aqualog.

```r
files <- list.files(
  path = "raw_data/170326",
  full.names = TRUE,
  pattern = ".+\\.csv"
)

read_feem(path = files[1])
```

Read and combine multiple FEEM files:

```r
feem_dat <- files[1:7] %>%
  set_names() %>%
  map_dfr(~ read_feem(.x, truncate = TRUE), .id = "file")
```

> [!NOTE]
> Replace the file path with the location of your local FEEM data directory. Replace the 1:7 with the range of desired FEEM files in your folder.

## Calculating HIX and BIX

The excitation and emission wavelength pairs used in older FEEM software differ slightly from the current CWRS Aqualog instrument. Use the modified code below when calculating HIX and BIX values.

```r
calculate_indices <- function(x)
{
  bix <- NULL
  hix <- NULL
  
  x %>% mutate(
    bix_numerator   = .data$excitation == 311 & .data$emission == 380.329,
    bix_denominator = .data$excitation == 311 & .data$emission == 431.034,
    
    hix_numerator   = .data$excitation == 254 & .data$emission >= 434 & .data$emission <= 480,
    hix_denominator = .data$excitation == 254 & .data$emission >= 300 & .data$emission <= 344
  ) %>%
    
    summarize(
      bix = .data$intensity[.data$bix_numerator] /
        .data$intensity[.data$bix_denominator],
      
      hix = sum(.data$intensity[.data$hix_numerator]) /
        sum(.data$intensity[.data$hix_denominator])
    ) %>%
    
    pivot_longer(c(bix, hix), names_to = "param")
}
```

## Organizing FEEM Data

Using a consistent file naming convention improves downstream analysis and plotting.

Example naming format:

```text
ALM11 020226.csv
```

Where:

* `ALM11` = 11 mg/L alum dose
* `020226` = sampling date (DDMMYY)

The following workflow converts FEEM filenames into a readable metadata table.

```r
regions_table <- feem_dat %>%
  group_by(file) %>%
  nest() %>%
  ungroup() %>%
  mutate(
    regions = map(data, integrate_regions),
    
    file_name = str_extract(file, "[^/]+$"),
    
    sample_name = str_extract(file_name, "(?i)(ALM\\d+|raw|cw)"),
    
    date = as.Date(
      str_extract(file_name, "\\d{6}"),
      format = "%d%m%y"
    ),
    
    alm_conc = case_when(
      str_detect(sample_name, regex("^ALM", ignore_case = TRUE)) ~ 
        as.numeric(str_extract(sample_name, "\\d+")),
      
      str_detect(sample_name, regex("^raw$", ignore_case = TRUE)) ~ 0,
      
      TRUE ~ NA_real_
    )
    
  ) %>%
  unnest(regions) %>%
  select(date, sample_name, alm_conc, region = name, integrated)
```
